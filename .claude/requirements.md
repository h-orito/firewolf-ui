# FIREWOLF UI v2 要件定義書

## プロジェクト概要

### 目的
既存の FIREWOLF UI v1（Nuxt.js 2.x + Vue.js 2.x）を現代的な技術スタックで刷新し、保守性、パフォーマンス、開発体験を向上させる。

### 背景
- v1 は Nuxt.js 2.x + Vue.js 2.x + TypeScript で構築されており、技術スタックの古さによる保守性の問題
- モダンな開発体験とパフォーマンスの向上が必要
- 既存の機能を維持しながら、技術的負債を解消する必要がある

## 機能要件

### 基本機能
v1で実装されている以下の機能をすべて移行する：

1. **認証機能**
   - Firebase Authentication を使用したユーザー認証
   - ログイン・ログアウト機能

2. **村一覧・村詳細**
   - 村一覧表示
   - 村詳細情報表示
   - 村への参加・観戦機能

3. **村作成機能**
   - 村の設定（キャラチップ、組織、制限時間など）
   - 村作成フォーム

4. **ゲーム機能**
   - メッセージ送信（発言、独り言、囁き等）
   - 投票機能
   - 能力実行
   - コミット機能
   - キャラクター変更

5. **管理機能**
   - 村管理者機能
   - デバッグ機能
   - 村設定変更

6. **その他**
   - プレイヤー統計
   - ルール説明
   - FAQ
   - リリースノート
   - キャラチップ一覧・詳細
   - 利用規約・プライバシーポリシー

### 新機能要件
- より良いUX/UIの提供
- パフォーマンスの向上
- モバイル対応の強化
- アクセシビリティの向上
- v1のトップページ画像を継続使用
- テストサーバー警告文の削除（本番環境での運用を想定）

#### トップページの改善
- トップページの紹介文を「FIREWOLFは長期人狼が無料で遊べるサービスです」に変更
- トップページ画像の右側に赤茶系のtext-shadowをかけつつサイト名（FIREWOLF）を画像上に表示
- トップページに表示する村はプロローグ、進行中、エピローグのもので固定とする
- お知らせ内容は v1 からコピーして、最新のお知らせ以外は `/release-note` で表示する
- トップページ画像は横いっぱいに表示する
- トップページの「チャット形式で気軽に人狼ゲームを楽しめます。初心者からエキスパートまで、誰でも参加できる多彩な村が開催中です。」の文章は不要

#### 村一覧UIの改善
- 開催中の村一覧では、表示する村の状態選択機能を削除し、プロローグ・進行中・エピローグの村のみを固定表示する
- ユーザーがフィルター設定を変更できないようにして、シンプルなUIを提供する

#### 村カード情報の改善
- **更新時刻の表示**: 開始予定の代わりに「更新 {日時}」を表示し、最新の日変更時刻を使用する
- **発言可能時間の表示**: v1と同様に発言可能時間を表示し、沈黙時間がある場合は時間範囲と総時間を併記する
- **ダミー役欠けの有無**: 村設定に基づいてダミー役欠けの有無を表示する
- **年齢制限の表示**: 年齢制限がある場合のみ項目として表示し、タグからR指定を検出する

#### トップページお知らせ機能の改善
- **お知らせ表示の拡充**: v1からお知らせ内容をコピーして最新のもの以外は `/release-note` ページへのリンクを提供する
- **過去のお知らせ**: v1から過去のお知らせ内容を移行して適切に表示する

#### UIテーマカラーの統一
- **ボタン系テーマカラーの統一**: 一貫性のある色彩設計でユーザビリティを向上させる
  - primary: 青系（メイン操作）
  - danger: 赤系（危険な操作、削除等）
  - success: 緑系（成功、完了等）
  - warning: 黄色系（注意、警告等）

#### 追加画面の実装
- キャラチップ一覧画面の追加（キャラクター素材集の表示）
- キャラチップ詳細画面の追加（個別キャラクター情報の表示）
- 利用規約ページの実装（v1と同じ内容）
- プライバシーポリシーページの実装（v1と同じ内容）

#### ファビコンの設定
- v1からファビコンをコピーして使用
- 統一性のあるブランディングを維持

#### UIコンポーネントの改善
- **Font Awesome アイコンの使用**: アイコン表示にFont Awesomeライブラリを使用
- **ボタンアイコンの統一設計**: ボタンには基本的にFont Awesomeアイコンを付加するデザインガイドライン
- **ボタンスタイルの統一**: 特別な場合を除いてsuccessボタンは使用せず、primaryボタンを使用
- **見出しスタイルの統一**: 見出し項目の大きさを統一し、CSSまたはコンポーネントで管理
- **共通ヘッダーの改善**:
  - FIREWOLFタイトルが画面中央に配置されるよう調整（現在少し右にずれている）
  - Google ログイン、Twitter ログインボタンの色調整（現在白くなって文字が読めない状態）
  - 「トップ」リンクの削除（サイト名リンクが実質トップへのリンクのため）
  - ログイン状態表示をヘッダー内に移動:
    - ログアウト機能
    - マイページリンク
    - 他SNSアカウント連携モーダル
  - ユーザーインターフェースの改善:
    - ログイン時: ユーザー名をクリックでアコーディオン展開（マイページ、ログアウト、他SNS連携等）
    - 未ログイン時: 「ログイン」ボタンクリックでTwitter/Googleログイン選択モーダル表示
- **共通フッターの改善**:
  - 紹介文「人狼ゲームが無料で遊べるWebサービスです。オンラインで手軽に人狼ゲームを楽しむことができます。」を削除
  - TwitterリンクをX（旧Twitter）の新URL（https://x.com/ort_dev）に変更
  - 著作権表記を「2020-」に変更
  - 「このサイトについて」をFIREWOLF項目に移動
  - 投げ銭項目の追加（v1のmodal-kampa.vueを参考に実装）
  - 「サイト情報」セクションを「Contact」に変更
  - 投げ銭機能をContact項目に移動
- **利用規約・プライバシーポリシーの修正**:
  - 現在リンク切れになっている利用規約・プライバシーポリシーページを修正
  - v1と同じ内容を表示するよう実装

#### トップページの改善（追加要件）
- **参加している村セクションの追加**: 「開催中の村」の上に「参加している村」セクションを追加

#### ログイン機能の改善
- **ログインモーダルの注意文追加**: ログインモーダルにv1のmodal-signin.vueの注意文を掲載
  - 「既存のアカウントに別のログイン方法を紐付けたい場合は、まず既存のアカウントでログインしてください。」
  - 「ログインしたことがあるアカウントは後から追加で紐付けることができないので、ご注意ください。」
  - 赤色のテキストで注意を促す表示

## 非機能要件

### パフォーマンス要件
- 初期読み込み時間の短縮（v1比で30%改善目標）
- Core Web Vitals の改善
- バンドルサイズの最適化

### 保守性要件
- TypeScript の厳密な型定義
- コンポーネントベースの設計
- テストカバレッジの向上
- ESLint + Prettier による一貫性のあるコード品質

### 対応ブラウザ
- Chrome, Firefox, Safari, Edge の最新版
- モバイルブラウザ対応

## 技術要件

### フロントエンド技術スタック
- **フレームワーク**: Next.js 15.x
- **言語**: TypeScript 5.x
- **UIライブラリ**: React 19.x
- **スタイリング**: Tailwind CSS
- **状態管理**: Zustand
- **データフェッチング**: TanStack Query (React Query)
- **API通信**: openapi-fetch
- **パッケージマネージャー**: pnpm

### 開発ツール
- **Linter**: ESLint + Prettier
- **型チェック**: TypeScript compiler
- **API型生成**: openapi-typescript

### インフラ・デプロイ
- **デプロイ環境**: Kubernetes クラスタ
- **コンテナレジストリ**: GitHub Container Registry (ghcr.io)
- **デプロイ戦略**: 
  - masterブランチへのpush時に自動でDocker imageをビルド
  - ghcr.ioへpush
  - Kubernetesマニフェストは別途管理
- **Dockerイメージ要件**:
  - マルチステージビルドによる最適化
  - 本番用の最小限のイメージサイズ
  - Node.js 22ベース
- **SSR/ISR対応**:
  - Next.jsのServer-Side Rendering (SSR)とIncremental Static Regeneration (ISR)をサポート
  - サーバーアクションの実行をサポート

## 制約事項

### 技術制約
- 既存のバックエンドAPI仕様との互換性を保つ
- Firebase Authentication の継続使用
- 既存のドメイン・インフラでのデプロイ
- `/firewolf` をベースパスとして使用（Next.jsのbasePath設定）

### スケジュール制約
- 段階的な移行を前提とした開発
- v1 との並行運用期間を考慮

### リソース制約
- 既存の開発リソース内での実装
- 外部ライブラリの選定は慎重に行う

## 移行要件

### データ移行
- ユーザーデータ、村データはバックエンド側で管理されているため、フロントエンド移行による影響はなし

### 段階的移行
1. 基本機能（ホーム、村一覧、村詳細）の移行
2. 認証機能の移行
3. 村作成・ゲーム機能の移行
4. 管理機能・その他機能の移行

- Core Web Vitals の改善
- バンドルサイズの削減
- 開発生産性の向上（TypeScriptの型安全性）

### 定性的指標
- ユーザビリティの向上
- 開発者体験の向上
