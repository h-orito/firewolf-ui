# FIREWOLF UI v2 要件定義書

## プロジェクト概要

### 目的

既存の FIREWOLF UI v1（Nuxt.js 2.x + Vue.js 2.x）を現代的な技術スタックで刷新し、保守性、パフォーマンス、開発体験を向上させる。

### 背景

- v1 は Nuxt.js 2.x + Vue.js 2.x + TypeScript で構築されており、技術スタックの古さによる保守性の問題
- モダンな開発体験とパフォーマンスの向上が必要
- 既存の機能を維持しながら、技術的負債を解消する必要がある

## 機能要件

### 基本機能

v1 で実装されている以下の機能をすべて移行する：

1. **認証機能**

   - Firebase Authentication を使用したユーザー認証
   - ログイン・ログアウト機能

2. **村一覧・村詳細**

   - 村一覧表示
   - 村詳細情報表示
   - 村への参加・観戦機能

3. **村作成機能**

   - 村の設定（キャラチップ、組織、制限時間など）
   - 村作成フォーム

4. **ゲーム機能**

   - メッセージ送信（発言、独り言、囁き等）
   - 投票機能
   - 能力実行
   - コミット機能
   - キャラクター変更

5. **管理機能**

   - 村管理者機能
   - デバッグ機能
   - 村設定変更

6. **その他**
   - プレイヤー統計
   - ルール説明
   - FAQ
   - リリースノート
   - キャラチップ一覧・詳細
   - 利用規約・プライバシーポリシー

### 新機能要件

- より良い UX/UI の提供
- パフォーマンスの向上
- モバイル対応の強化
- アクセシビリティの向上
- v1 のトップページ画像を継続使用
- テストサーバー警告文の削除（本番環境での運用を想定）

#### UI テーマカラーの統一

- **ボタン系テーマカラーの統一**: 一貫性のある色彩設計でユーザビリティを向上させる
  - primary: 青系（メイン操作）
  - danger: 赤系（危険な操作、削除等）
  - success: 緑系（成功、完了等）
  - warning: 黄色系（注意、警告等）

#### ファビコンの設定

- v1 からファビコンをコピーして使用
- 統一性のあるブランディングを維持

#### UI コンポーネントの改善

- **Font Awesome アイコンの使用**: アイコン表示に Font Awesome ライブラリを使用
- **ボタンアイコンの統一設計**: ボタンには基本的に Font Awesome アイコンを付加するデザインガイドライン
- **ボタンスタイルの統一**: 特別な場合を除いて success ボタンは使用せず、primary ボタンを使用
- **見出しスタイルの統一**: 見出し項目の大きさを統一し、CSS またはコンポーネントで管理
- **コンポーネントディレクトリ構造の整理**: components 直下のページ用コンポーネントを components/pages ディレクトリに移動してわかりやすく整理
- **見出し要素の統一**: h1、h2 等の直接使用を避け、heading.tsx コンポーネントで定義したスタイルを統一使用
- **共通ヘッダーの改善**:
  - FIREWOLF タイトルが画面中央に配置されるよう調整（現在少し右にずれている）
  - Google ログイン、Twitter ログインボタンの色調整（現在白くなって文字が読めない状態）
  - 「トップ」リンクの削除（サイト名リンクが実質トップへのリンクのため）
  - ログイン状態表示をヘッダー内に移動:
    - ログアウト機能
    - マイページリンク
    - 他 SNS アカウント連携モーダル
  - ユーザーインターフェースの改善:
    - ログイン時: ユーザー名をクリックでアコーディオン展開（マイページ、ログアウト、他 SNS 連携等）
    - 未ログイン時: 「ログイン」ボタンクリックで Twitter/Google ログイン選択モーダル表示
- **共通フッターの改善**:
  - 紹介文「人狼ゲームが無料で遊べる Web サービスです。オンラインで手軽に人狼ゲームを楽しむことができます。」を削除
  - Twitter リンクを X（旧 Twitter）の新 URL（https://x.com/ort_dev）に変更
  - 著作権表記を「2020-」に変更
  - 「このサイトについて」を FIREWOLF 項目に移動
  - 投げ銭項目の追加（v1 の modal-kampa.vue を参考に実装）
  - 「サイト情報」セクションを「Contact」に変更
  - 投げ銭機能を Contact 項目に移動
- **利用規約・プライバシーポリシーの修正**:
  - 現在リンク切れになっている利用規約・プライバシーポリシーページを修正
  - v1 と同じ内容を表示するよう実装

## 非機能要件

### パフォーマンス要件

- 初期読み込み時間の短縮（v1 比で 30%改善目標）
- Core Web Vitals の改善
- バンドルサイズの最適化

### 保守性要件

- TypeScript の厳密な型定義
- コンポーネントベースの設計
- テストカバレッジの向上
- ESLint + Prettier による一貫性のあるコード品質

### 対応ブラウザ

- Chrome, Firefox, Safari, Edge の最新版
- モバイルブラウザ対応

## 技術要件

### フロントエンド技術スタック

- **フレームワーク**: Next.js 15.x
- **言語**: TypeScript 5.x
- **UI ライブラリ**: React 19.x
- **スタイリング**: Tailwind CSS
- **状態管理**: Zustand
- **データフェッチング**: TanStack Query (React Query)
- **API 通信**: openapi-fetch
- **パッケージマネージャー**: pnpm

### 開発ツール

- **Linter**: ESLint + Prettier
- **型チェック**: TypeScript compiler
- **API 型生成**: openapi-typescript

### インフラ・デプロイ

- **デプロイ環境**: Kubernetes クラスタ
- **コンテナレジストリ**: GitHub Container Registry (ghcr.io)
- **デプロイ戦略**:
  - master ブランチへの push 時に自動で Docker image をビルド
  - ghcr.io へ push
  - Kubernetes マニフェストは別途管理
- **Docker イメージ要件**:
  - マルチステージビルドによる最適化
  - 本番用の最小限のイメージサイズ
  - Node.js 22 ベース
- **SSR/ISR 対応**:
  - Next.js の Server-Side Rendering (SSR)と Incremental Static Regeneration (ISR)をサポート
  - サーバーアクションの実行をサポート

## 制約事項

### 技術制約

- 既存のバックエンド API 仕様との互換性を保つ
- Firebase Authentication の継続使用
- 既存のドメイン・インフラでのデプロイ
- `/firewolf` をベースパスとして使用（Next.js の basePath 設定）

### スケジュール制約

- 段階的な移行を前提とした開発
- v1 との並行運用期間を考慮

### リソース制約

- 既存の開発リソース内での実装
- 外部ライブラリの選定は慎重に行う

## 移行要件

### データ移行

- ユーザーデータ、村データはバックエンド側で管理されているため、フロントエンド移行による影響はなし

### 段階的移行

1. 基本機能（ホーム、村一覧、村詳細）の移行
2. 認証機能の移行
3. 村作成・ゲーム機能の移行
4. 管理機能・その他機能の移行

- スマホではハンバーガーメニュー形式
- 発言抽出機能による高度なフィルタリング
- リアルタイム更新（残り時間表示等）
- ユーザー設定の永続化（Cookie 保存）
- 個人抽出機能（別タブ/同一タブ選択可能）
- Google AdSense 広告の統合（サイドバー・メインコンテンツ）
- 高度なメッセージ装飾機能（デコレーション・アンカー）
- 包括的なアクション機能（発言・入村・見学・能力行使等）
