## 優先順位

1. **最優先**: bugs.mdに記載されたバグ修正
2. **高**: features.mdに記載された新機能要望
3. **中**: リファクタリングやパフォーマンス改善
4. **低**: ドキュメント更新

## 注意事項

- 各タスクは適切な粒度でコミットする
- 実装後は必ず lint、format、type-check を実行
- API の型は自動生成されたものを使用
- ドキュメントは実装と並行して更新
- v1 との互換性を意識（URL 構造、API 形式等）
