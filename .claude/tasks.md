## 注意事項

### 開発プロセス
- 各タスクは適切な粒度でコミットする
- 実装後は必ず lint、format、type-check を実行
- API の型は自動生成されたものを使用
- ドキュメントは実装と並行して更新
- v1 との互換性を意識（URL 構造、API 形式等）

### 状態管理の重要原則
- **フィルタ・日付状態は永続化しない**（別タブ独立性のため）
- **ユーザー設定・メモのみCookie保存**（永続化が必要なもののみ）
- **各タブはデフォルト状態から開始**（最新日・全フィルタON）
- **URLパラメータでの初期化対応**（個人抽出・日付指定）

### パフォーマンス要件
- **リアルタイム更新間隔**：残り時間1秒、発言データ30秒、村情報60秒
- **サーバー負荷軽減**を最優先に設計
- **仮想化・メモ化**を適切に適用

### UI/UX要件
- **システムメッセージの正確な色設定**（message.md準拠）
- **アクセシビリティ対応**（WCAG 2.1 AA準拠）
- **別タブでの柔軟な利用**を前提とした設計
