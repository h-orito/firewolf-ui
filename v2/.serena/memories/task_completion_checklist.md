# タスク完了時チェックリスト

## 必須実行手順（品質チェック）

### 1. コード品質チェック

```bash
# ESLintでコードの品質をチェック（エラー・警告をすべて解消）
pnpm lint

# Prettierでコードをフォーマット
pnpm format

# TypeScriptの型チェック（型エラーをすべて解消）
pnpm type-check
```

### 2. テスト実行

```bash
# 単体テストが通ることを確認
pnpm test

# 重要な機能の場合は統合テストも実行
pnpm test:e2e
```

### 3. ビルドテスト

```bash
# 本番ビルドが正常に完了することを確認
pnpm build
```

## タスク管理更新

### 1. タスクステータス更新

- `.claude/tasks.md` の該当タスクを「完了」に更新
- 必要に応じて新しいタスクを追加

### 2. バグ修正の場合

- `.claude/bugs.md` から対応済み項目を削除
- 修正内容をユーザーに確認を依頼

## コミット前確認

### 1. 変更内容の確認

```bash
# 変更されたファイルを確認
git status

# 変更差分を確認
git diff
```

### 2. コミットメッセージ規約

- **言語**: 日本語
- **絵文字**: 使用しない
- **プレフィックス**:
  - `feat:` 新機能
  - `fix:` バグ修正
  - `refactor:` リファクタリング
  - `docs:` ドキュメント更新
  - `style:` コードスタイルの変更
  - `test:` テストの追加・修正
  - `chore:` ビルドプロセスやツールの変更

### 3. コミット単位

- 機能単位またはファイル単位で適切に分割
- 一度に大量の変更をコミットしない

## ドキュメント更新

### 1. 実装ドキュメント

- 必要に応じて `doc/v2/` 配下のドキュメントを更新

### 2. 設計書更新

- 大きな変更の場合は `.claude/designs.md` を更新

## フロー継続

### 1. 新しいタスクの確認

- 完了後、必ず再度 `.claude/bugs.md`, `.claude/features.md`, `.claude/tasks.md` を確認
- 新しい不具合や機能要望がないかチェック

### 2. 継続的な改善

- パフォーマンスやユーザビリティの改善点があれば記録
