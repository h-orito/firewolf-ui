# v2 プロジェクト要件定義

## 1. プロジェクト概要

FIREWOLF UI v2 は、既存の v1 プロジェクトを刷新し、より良いユーザー体験とメンテナンス性を提供する人狼ゲーム Web サービスのフロントエンドです。

## 2. 機能要件

### 2.1 基本機能

- ユーザー認証・認可
- 村の作成・参加・管理
- ゲーム進行の管理
- チャット・メッセージ機能
- プレイヤー記録の管理

### 2.2 村作成画面のルール設定改善

#### 2.2.1 設定項目の改善

1. **記名投票設定**
   - 現在の「投票公開」を「記名投票」に名称変更
   - 設定値：公開 / 非公開
2. **見学設定**
   - 「見学可能」の初期値を「なし」に変更
   - 設定値：可能 / なし
3. **突然死設定**
   - 「突然死あり」項目を新規追加
   - 初期値：あり
   - 設定値：あり / なし
4. **時短希望設定**
   - 現在の「コミット可能」を「時短希望可能」に名称変更
   - 設定値：可能 / 不可能
5. **独り言設定**
   - 「独り言可能」項目を削除
6. **連続護衛設定**
   - 「連続護衛可能」項目を新規追加
   - 初期値：あり
   - 設定値：可能 / 不可能

### 2.3 発言制限設定の詳細化

#### 2.3.1 発言種別と個別制限設定

各発言種別に対して個別の回数制限と文字数制限を設定可能にする：

1. **通常発言**
   - 回数制限：1-1000（最小値：1）
   - 文字数制限：1-1000
   - 初期値：回数 20回、文字数 400文字

2. **人狼の囁き**
   - 回数制限：0-1000
   - 文字数制限：1-1000
   - 初期値：回数 20回、文字数 400文字

3. **共鳴発言**
   - 回数制限：0-1000
   - 文字数制限：1-1000
   - 初期値：回数 20回、文字数 400文字

4. **恋人発言**
   - 回数制限：0-1000
   - 文字数制限：1-1000
   - 初期値：回数 20回、文字数 400文字

5. **死者の呻き**
   - 回数制限：0-1000
   - 文字数制限：1-1000
   - 初期値：回数 5回、文字数 200文字

6. **独り言**
   - 回数制限：0-1000
   - 文字数制限：1-1000
   - 初期値：回数 20回、文字数 400文字

7. **見学発言**
   - 回数制限：0-1000
   - 文字数制限：1-1000
   - 初期値：回数 20回、文字数 400文字

#### 2.3.2 制限値の範囲

- **回数制限**
  - 通常発言：1-1000
  - その他の発言種別：0-1000
- **文字数制限**
  - 全発言種別：1-1000

## 3. 非機能要件

### 3.1 パフォーマンス

- ページ読み込み時間：3秒以内
- リアルタイム通信の遅延：1秒以内

### 3.2 ユーザビリティ

- レスポンシブデザイン対応
- アクセシビリティ対応（WCAG 2.1 AA準拠）

### 3.3 セキュリティ

- HTTPS通信の強制
- XSS、CSRF攻撃の対策
- 適切な認証・認可の実装

### 3.4 保守性

- TypeScript による型安全性の確保
- テストカバレッジ 80% 以上
- ESLint、Prettier によるコード品質の維持

## 4. 技術的制約

### 4.1 技術スタック

- Framework: Next.js 14+
- Language: TypeScript
- UI Library: Tailwind CSS
- State Management: Zustand
- Testing: Jest, Playwright

### 4.2 ブラウザサポート

- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

## 5. 運用要件

### 5.1 デプロイメント

- CI/CD パイプラインによる自動デプロイ
- ステージング環境での事前検証

### 5.2 監視・ログ

- エラーログの収集と分析
- パフォーマンス監視

## 6. 移行要件

### 6.1 データ移行

- v1 からのユーザーデータ移行
- 既存の村データの継続利用

### 6.2 URL互換性

- 主要なページのURL構造の維持
- リダイレクト設定による互換性確保
