# 発言抽出機能仕様

## 概要

発言一覧に表示する内容を細かくフィルタリングできる機能です。画面最下部の「発言抽出」ボタンから専用モーダルを開いて設定を行います。

## モーダル構成

### 基本レイアウト

- **モーダルタイトル**: 「発言抽出」
- **サイズ**: 大きめのモーダル（内容量に応じて可変）
- **操作ボタン**: 「適用」「リセット」「キャンセル」

## フィルタ項目

### 1. 発言種別フィルタ

#### 一括操作ボタン

- **全て ON**: すべての発言種別を選択
- **全て OFF**: すべての発言種別を非選択
- **反転**: 現在の選択状態を反転

#### 選択可能な発言種別

複数選択可能（チェックボックス形式）。デフォルトは全て ON：

1. **通常発言** - 一般的な発言
2. **独り言** - 他プレイヤーに見えない個人の思考
3. **秘話** - 特定の相手との秘密の会話
4. **村建て** - 村作成者からのお知らせ
5. **囁き** - 人狼同士の秘密会話
6. **共鳴** - 共鳴者同士の会話
7. **恋人** - 恋人同士の会話
8. **墓下見学** - 死亡者・見学者の会話
9. **アクション** - ゲーム内アクション
10. **公開システム** - システムからの公開メッセージ
11. **非公開システム** - システムからの個人向けメッセージ

#### UI 表示

- チェックボックスリスト形式
- 各項目の説明も併記

### 2. 発言者フィルタ

#### 一括操作ボタン

- **全て ON**: すべての参加者を選択
- **全て OFF**: すべての参加者を非選択
- **反転**: 現在の選択状態を反転

#### 参加者選択

- **レイアウト**: 2 列表示で効率的に表示
- **デフォルト**: 全て ON
- **選択形式**: チェックボックス

#### 参加者表示項目

各参加者について以下を表示：

- **キャラアイコン**: API 返却サイズの 1/2 で表示
- **キャラ名**: `[{short_name}] {name}` 形式
- **発言数**: 選択中の日での発言数（例：「12 回」）

### 3. 宛先フィルタ

#### 一括操作ボタン

- **全て ON**: すべての参加者を選択
- **全て OFF**: すべての参加者を非選択
- **反転**: 現在の選択状態を反転
- **自分宛**: 自分が宛先に含まれる発言のみ選択

#### 宛先選択

アンカーやメンション等で宛先が指定された発言をフィルタ：

- **レイアウト**: 参加者と同様に 2 列表示
- **デフォルト**: 全て ON
- **「自分宛」機能**: ログイン中のユーザーが宛先の発言を一括選択

#### 参加者表示項目

発言者フィルタと同一：

- **キャラアイコン**: API 返却サイズの 1/2 で表示
- **キャラ名**: `[{short_name}] {name}` 形式

### 4. キーワードフィルタ

#### 入力フォーム

- **入力欄**: テキスト入力フィールド
- **プレースホルダー**: 「スペース区切り」
- **複数キーワード**: 半角スペースまたは全角スペースで区切って複数指定可能

#### 検索動作

- **OR 検索**: 指定したキーワードのいずれかが含まれる発言を抽出
- **部分一致**: キーワードの部分一致で検索
- **大文字小文字**: 区別しない（日本語での運用を想定）

#### 使用例

- 「人狼 投票」 → 「人狼」と「投票」のどちらかを含む発言
- 「占い師」 → 「占い師」を含む発言

## 動作仕様

### フィルタ適用

1. **適用ボタン**: 設定内容を適用してモーダルを閉じる
2. **発言一覧更新**: 条件に合致する発言のみ表示

### 設定保存

- **セッション保持**: モーダルを閉じても設定を保持

### リセット機能

- **リセットボタン**: すべての設定をデフォルト状態（全て ON）に戻す
- **確認ダイアログ**: リセット前に確認を表示

### アクセシビリティ対応

- **キーボードナビゲーション**: Tab/Enter/Space での操作
- **スクリーンリーダー**: 適切なラベルと aria 属性
- **フォーカス管理**: モーダル内でのフォーカストラップ

### パフォーマンス考慮

- **大人数村対応**: 参加者リストの仮想化
- **リアルタイムフィルタ**: デバウンス処理で負荷軽減
- **メモリ効率**: 不要なレンダリングの防止

## 技術実装

### 状態管理

```typescript
interface MessageFilterState {
  // 発言種別フィルタ
  messageTypes: {
    normal: boolean;
    monologue: boolean;
    secret: boolean;
    // ... その他の種別
  };

  // 発言者フィルタ
  speakers: string[]; // participant_idの配列

  // 宛先フィルタ
  targets: string[]; // participant_idの配列

  // キーワードフィルタ
  keywords: string[];
}
```

### API 連携

- フィルタリングはフロントエンド側で実行
- API から取得した全発言データをクライアントサイドでフィルタ

### コンポーネント構成

```
MessageFilter/
├── index.tsx                    # モーダルメイン
├── MessageTypeFilter.tsx        # 発言種別フィルタ
├── SpeakerFilter.tsx           # 発言者フィルタ
├── TargetFilter.tsx            # 宛先フィルタ
├── KeywordFilter.tsx           # キーワードフィルタ
├── BulkActions.tsx             # 一括操作ボタン
└── FilterPreview.tsx           # プレビュー表示（オプション）
```

## 追加仕様

### URL 連携機能

- **filterId パラメータ**: URL に`?filterId={participant_id}`が指定された場合
- **自動フィルタ適用**: 該当プレイヤーの発言のみ抽出した状態で画面を初期化
- **個人抽出機能**: 左側メニューの個人抽出と連携

### エクスポート機能（将来拡張）

- **設定の保存**: よく使うフィルタ設定を保存
- **設定の共有**: フィルタ設定を URL で共有可能
