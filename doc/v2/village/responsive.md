# レスポンシブ対応仕様

## 概要

村画面のレスポンシブデザイン仕様です。PC・タブレット・スマートフォンの各デバイスで最適な表示と操作性を提供します。

## ブレークポイント

Tailwind CSS の標準ブレークポイントを使用：

- **sm**: 640px以上（タブレット縦向き）
- **md**: 768px以上（タブレット横向き）  
- **lg**: 1024px以上（デスクトップ）
- **xl**: 1280px以上（大画面）

## デバイス別レイアウト

### スマートフォン（～767px）

#### レイアウト構成
- **左側メニュー**: 通常は非表示、ハンバーガーメニューで表示
- **メインコンテンツ**: 画面幅いっぱいに表示
- **ナビゲーション**: 最下部固定メニューに集約

#### 左側メニュー（スマホ）
- **表示方法**: スライドオーバー形式
- **開閉操作**: 
  - 開く: 最下部メニューのハンバーガーボタン（☰）
  - 閉じる: オーバーレイタップ、×ボタン、スワイプ
- **アニメーション**: スライドイン/アウト
- **背景**: 半透明のオーバーレイ

#### 最下部固定メニュー（スマホ）
左から右に以下のボタンを配置：

1. **ハンバーガーメニュー（☰）** - 左側メニューを開く
2. **更新（🔄）** - 最新発言表示
3. **抽出（🔍）** - 発言抽出モーダル
4. **上へ（↑）** - 最上部へスクロール
5. **下へ（↓）** - 最下部へスクロール

中央に**残り時間表示**を配置。

### タブレット（768px～1023px）

#### レイアウト構成
- **左側メニュー**: 縮小表示（折りたたみ対応）
- **メインコンテンツ**: 残りの画面幅を使用
- **操作**: タッチとマウス両方に対応

#### 左側メニュー（タブレット）
- **幅**: PCの70%程度に縮小
- **折りたたみ**: 長い項目は省略表示
- **参加者リスト**: 2列→1列に変更

### PC・デスクトップ（1024px以上）

#### レイアウト構成
- **左側メニュー**: 固定幅で常時表示
- **メインコンテンツ**: 残りの画面幅を使用
- **操作**: マウス・キーボード最適化

## コンポーネント別対応

### 1. 左側メニュー

#### 参加者リスト
- **PC**: 1列表示、アイコンとテキストを横並び
- **タブレット**: 1列表示、省略形式
- **スマホ**: 1列表示、コンパクト形式

#### ユーザー設定
- **PC**: 詳細設定画面
- **スマホ**: アコーディオン形式で省スペース表示

#### メモ機能
- **PC**: サイドバー内に常時表示
- **スマホ**: 折りたたみ表示

### 2. メインコンテンツ

#### 最上部固定メニュー
- **PC**: 3つのボタンを横並び配置
- **スマホ**: 省略または縦配置

#### 日付リンク
- **PC**: 横スクロールなしで全日付表示
- **スマホ**: 横スクロール対応

#### 発言一覧
- **PC**: 1ページあたり多めの発言数
- **スマホ**: 1ページあたり少なめの発言数で読みやすさを重視

### 3. 発言抽出モーダル

#### モーダルサイズ
- **PC**: 大きめのモーダル（800px幅程度）
- **スマホ**: ほぼ全画面表示

#### 参加者選択
- **PC**: 2列表示
- **タブレット**: 2列表示（狭め）
- **スマホ**: 1列表示

#### 操作ボタン
- **PC**: 横並び配置
- **スマホ**: 縦並びまたは固定配置

## タッチ操作対応

### タップ可能領域
- **最小タップサイズ**: 44px × 44px
- **適切な間隔**: ボタン間に8px以上の余白

### スワイプ操作
- **左側メニュー**: 
  - 右スワイプで開く
  - 左スワイプまたはオーバーレイタップで閉じる
- **発言一覧**: 
  - 上下スワイプでスクロール
  - 左右スワイプで日付切り替え（検討）

### ピンチズーム
- **メタビューポート**: `user-scalable=yes` で拡大可能
- **最大拡大率**: 2倍程度に制限

## パフォーマンス最適化

### 画像の最適化
- **レスポンシブ画像**: 複数サイズの画像を用意
- **遅延読み込み**: キャラアイコンの効率的な読み込み

### レンダリング最適化
- **仮想スクロール**: スマホでの長い発言リスト対応
- **条件付きレンダリング**: デバイスサイズに応じた表示内容の調整

## 実装技術

### CSS設計

```css
/* レスポンシブコンテナ */
.village-container {
  @apply flex;
  
  @screen sm {
    /* タブレット以上 */
  }
  
  @screen lg {
    /* PC以上 */
  }
}

/* 左側メニュー */
.sidebar {
  @apply hidden lg:block lg:w-80;
  
  /* スマホ: オーバーレイ表示 */
  &.mobile-open {
    @apply fixed inset-y-0 left-0 z-50 w-80 bg-white;
  }
}

/* メインコンテンツ */
.main-content {
  @apply flex-1 min-w-0;
  
  @screen lg {
    @apply ml-80;
  }
}
```

### JavaScript対応

```typescript
// デバイス判定
const isMobile = () => window.innerWidth < 768;
const isTablet = () => window.innerWidth >= 768 && window.innerWidth < 1024;
const isDesktop = () => window.innerWidth >= 1024;

// メニュー表示制御
const [isSidebarOpen, setIsSidebarOpen] = useState(false);

// レスポンシブ対応のuseEffect
useEffect(() => {
  const handleResize = () => {
    if (isDesktop()) {
      setIsSidebarOpen(true); // PCでは常時表示
    } else {
      setIsSidebarOpen(false); // モバイル・タブレットでは閉じる
    }
  };
  
  window.addEventListener('resize', handleResize);
  return () => window.removeEventListener('resize', handleResize);
}, []);
```

## アクセシビリティ

### フォーカス管理
- **モバイルメニュー**: 開いた時に適切な要素にフォーカス
- **モーダル**: フォーカストラップの実装

### キーボードナビゲーション
- **Escキー**: モーダルやメニューを閉じる
- **Tabキー**: 論理的な順序でのフォーカス移動

### スクリーンリーダー対応
- **aria-expanded**: メニューの開閉状態
- **aria-label**: ハンバーガーボタンの説明
